ICRL 2025#

Transfusion: Predict the Next Token and Diffuse Images with One Multi-Modal Model https://arxiv.org/pdf/2408.11039 https://zhuanlan.zhihu.com/p/720035369

这篇论文提出了一种名为 Transfusion 的新方法，旨在解决多模态生成模型中的核心矛盾：如何在一个统一的模型中高效地处理离散数据（文本）和连续数据（图像）

在多模态大模型（LMM）领域，目前主流的方法主要分为两类，但都有各自的缺陷：

“胶水”式拼接（Late Fusion / Tool Use）： 将预训练的 LLM（处理文本）和 Diffusion Model（处理图像）拼在一起。

缺点： 两个模型是独立的，无法真正实现深度的模态融合和理解。

全离散化（Quantization / Discrete Tokens）： 像 Chameleon 或 Google Gemini (部分版本) 那样，通过 VQ-VAE 将图像也量化成离散的 Token（类似于单词），然后像训练 LLM 一样训练。

缺点： 图像离散化会丢失信息（有损压缩），且为了描述一张图需要大量的 Token，计算成本高，扩展性差

Transfusion 的核心洞察： 不需要把图像强行变成离散的 Token。我们可以在同一个 Transformer 主干网络中，同时运行两套损失函数：对文本做“下一个词预测”，对图像做“扩散（Diffusion）去噪”

在Transfusion里面，文本还是按照语言模型的离散方式进行描述，但是图像则用连续的latent embedding来进行编码。具体来说和以往的自回归图像生成里面采用默认的VQ-VAE进行编码不一样，Transfusion采用了Stable Diffusion里面的VAE模型，即图片被编码为连续的特征序列，VQ-VAE和VAE的训练方式也不一样，VQ-VAE有什么comitment loss，VAE有KL-Loss。

和诸多多模态大模型一样，Transfusion也是采用了一个Transformer模型，Transformer输入连续的特征序列，同时也输出和输入处在统一特征空间的特征序列。为了达到这样的目的，文本采用的是embedding matrix，将代表每一个token的证书映射为连续的向量，模型输出的连续的embedding向量也会通过这个embedding matrix映射到对应的token id，然后解码出文本。对于图片来说，则是用预训练的VAE的Encoder将图片转化为特征矩阵，然后将窗口大小为 的特征映射为一个特征向量，文章也尝试了两种方式，一种是简单的线性层，一种是U-Net的up-down模块。模型输出的图片特征序列通过VAE的Decoder解码出对应的图片。文中的输入图片分辨率为 ，通过VAE的Encoder之后，得到的特征Tensor为 ，和Latent Diffusion类似

模型的训练数据包含1T的token，文本和图像的比例为1:1。文中也提到，在用VAE的Encode将 的图片编码到 的Tensor之后，会将大小为 窗口内的tensor编码为一个token embedding，文中也尝试了不同的 值对模型性能的影响，如图Fig 5所示，也比较了简单的线性层和U-Net的Up-down模块那一个性能更优

字节同期的工作《Show-O: One Single Transformer To Unify Multimodal Understanding And Generation》也是尝试用一个Transformer做多模态的内容理解和生成，而且文本和图像的生成也是采取了不同的方式：文本是Next-Token Prediction，图像则是类似于MUSE和MaskGIT里面的Masked Image Modeling，不过文中叫做离散的扩散过程，但是和扩散过程还是有一些差异的

https://ac-dit.github.io/

AC-DiT 针对的是移动操作机器人（mobile manipulation，底盘 + 机械臂组合）的协调控制问题。

两个核心问题：

  1. 底盘运动会影响机械臂，但现有方法没有显式建模这种依赖关系，自由度多导致误差累积
  2. 任务不同阶段对视觉的需求不同（有时需要语义理解，有时需要精确空间几何），但现有方法对整个任务用同一套视觉处理

解决方法：

  1. Mobility-to-Body Conditioning：先提取底盘运动表示，再用它作为条件来预测整体动作——机械臂的控制显式受底盘影响，层次化建模。
  2. Perception-Aware Multimodal Adaptation：根据任务阶段动态调整 2D 图像和 3D 点云的融合权重——需要语义理解时多用 2D，需要空间精度时多用 3D 点云。

用的是 Diffusion Transformer (DiT) 作为基础架构，在仿真和真实机器人上都验证了效果

https://kolakivy.github.io/AFRO/ 港广科 动态装配 核心思路是自监督预训练一个动态感知的 3D 视觉表征，不需要动作标注或重建监督 https://kolakivy.github.io/ 佬

https://github.com/PKU-HMI-Lab/LIFT3D 仉尚航团队。北大HMI lab。Lift3D 解决的是：现有大规模预训练模型都是 2D 的，但机器人操作需要 3D 空间感知，直接用 2D 模型会丢失空间几何信息，而 3D 数据又很稀缺。

核心做法：

  1. 隐式 3D 增强：用 task-aware masked autoencoder，遮住与操作相关的 patch，然后重建深度信息——让模型"学会"空间感知，不需要显式 3D 数据。
  2. 显式 3D 增强：把点云的 3D 位置映射到 2D 模型的 positional embedding（通过虚拟平面投影），让原本只懂 2D 的模型直接处理点云输入。

结果：在仿真 benchmark 和真实 Franka 机械臂的 10 个操作任务上超过了当前 SOTA，不需要重新训练大模型，只是把 2D 基础模型"升维"了

cot-vla https://cot-vla.github.io/

  1. 指出现有VLA的缺陷：当前最先进的VLA（如RT-2、Octo等）本质上是学习从观察到动作的“直接映射”（端到端），缺少完成任务所需的中间推理步骤。这使得它们难以处理需要多步规划或因果理解的复杂操作任务。
  2. 提出核心创新方法（视觉CoT）：
    • 核心思想：让模型在执行动作之前，先自回归地预测未来的图像帧，将这些预测出的图像作为达到最终目标的“视觉子目标”或“中间推理步骤”。
    • 工作流程：模型不再是直接输出动作序列，而是先进行“视觉推理”（预测下一步应该看到什么画面），然后再根据这个推理出的视觉目标，生成一小段能达成该目标的动作序列。
  3. 构建并验证了CoT-VLA模型：
    • 他们实现了一个 70亿参数（7B） 的VLA模型，该模型具备同时理解和生成视觉标记（图像） 和动作标记的能力。
    • 通过在真实机器人和仿真基准（如CALVIN、RLBench） 中的大量操作任务实验，证明了加入视觉思维链推理后，模型在复杂操作任务上的性能显著优于传统VLA

VLA行业调研 https://pdf.dfcfw.com/pdf/H3_AP202507071704760168_1.pdf?1751961651000.pdf

我们通过对机器人 VLA 与汽车 VLA 进行对比分析以探究机器人VLA数据闭环构建中面临的突出问题，本质上汽车 VLA 与机器人 VLA 所面临的应用场景与任务的不同决定了二者拥有不同的商品属性和标准化程度。汽车可以被视为一种低自由度的特殊机器人，所面临的场景和任务是结构化场景下的单一任务，根本上决定了汽车是感知方式、输出控制、自由度、本体结构都相对标准化的耐用消费品，底层硬件结构的统一性和场景、任务单一性决定了其数据采集方式、所采集的数据、模型设计的标准化；而广义的机器人由于所面临的场景与任务需求多样化，更多偏向于非标的可选消费电子类产品，更需要在模型设计、本体结构标准化、数据采集效率、任务覆盖范围等多个条件中进行权衡。二者场景、任务的不同，决定了二者商品属性和标准化程度不同进而决定了二者上层结构的差异。我们尝试从数据、仿真环境、端侧算力等具体角度对比汽车VLA 与机器人VLA的差异性。整体来看，机器人的数据闭环构建难度远超车端，其工程部署还需要解决标准化、本体交互能力、模型闭环等重要问题，机器人 VLA 所面临的各种问题汇集在一起突出表现为当前还无法进行有效的数据收集进而构建完整数据闭环，而无法Scaling 的具身就无从谈起智能化，因此我们认为机器人 VLA 模型或智能化当前还处于前期探索阶段，相较于汽车VLA专注于工程化，机器人 VLA 更是一个科研问题

机器人 VLA 模型所需的数据更为复杂多样，核心原因在于机器人所面临的场景和任务更为多样化，泛化能力要求远高于汽车。1）应用场景与任务不同，自动驾驶的汽车可以被理解为一个特殊的机器人，其所面临的结构化道路场景和执行的驾驶任务都较为单一；机器人如人形机器人所面临的场景和任务非常丰富，部署场景从家庭、工厂等封闭式场合到公共服务场合等开放式场景，任务从家政服务到工厂务工，几乎能囊括人类日常生活的各方面，因此从训练数据的多样性角度，其数据需求远超汽车；2）模型能力要求不同，从应用场景和任务出发，车端 VLA 重点在于提升端侧推理效率以增强模型的动态博弈能力，更加注重感知数据输入和 2D 轨迹规划输出以及模型实时决策，其数据维度较低（可以理解为一个动作专用模型利用同构低维数据，强化学习反复迭代提升性能）；机器人VLA 则更为注重模型的泛化性，其所需求的数据除感知数据以外，更为关注本体与真实世界的物理交互数据（如力反馈、摩擦力数据等），且更多输出 3D 空间动作规划，所需数据的维度、复杂度更高。因此在假设完美完成一类任务所需数据规模差异不大的前提下，机器人所需的数据从多样性到复杂度相较于车端都有较大提升

Open VLA 为一个 7B 规模的 VLA 模型，能够在接收信息后预测7 维的机器人控制动作，其模型架构主要由输入模块、LLM 主干两部分构成

输入模块由两个视觉编码器、小型两层 MLP 投影器（用于映射）、Llama 语言编码器构成，其中视觉编码器部分包含预训练的 SigLIP 和 DinoV2 模型，输入图像patch分别通过两个编码器得到结果特征向量，相较于 CLIP 或仅 SigLIP 编码器，添加DinoV2 编码器有助于提升模型空间推理能力，更能理解物体间的准确位置关系。LLM 主干部分为一个 7B 规模的 Llama 2 大语言模型并在Open X-Embodiment 机器人数据集上进行了微调，接收图像、文本等多模态信息并进行推理输出action token。模型利用栅格法，将机器人连续的动作离散化分散到 256 个栅格中，每一个栅格可以被视作一个token。当机器人动作被处理为 token 序列后，VLA 模型即能够以标准的自回归方式预测下一个token为目标进行训练

ViLLA 架构的亮点在于对大规模互联网异构视频数据的高效利用。ViLLA架构由VLM＋MOE 组成，主要由 VLM、Latent Planner、Action Expert 三大模块协同工作，相较于传统的VLA，ViLLA 通过预测隐式动作标记，弥合图像-文本输入与机器人执行动作之间的差距，在真实世界的灵巧操作和长时任务方面表现较好，超过了彼时已有的开源SOTA模型；同时LAM的引入，使模型不仅能够从互联网数据中增强语义理解和场景理解能力，还能够直接从人类视频演示中进行轨迹动作知识的迁移，实现对不同互联网数据的高效利用

VLM（多模态大模型）：采用 InternVL-2B，接收多视角图像、力信号、语言输入等，进行通用场景感知和指令理解。 Latent Planner（隐式规划器）：LP 是 MoE 中的一组专家，其基于VLM的输出结果预测隐式动作标记以进行高层级动作规划。为了大规模利用互联网视频数据和训练Latent Planner，ViLLA 架构在云端引入了 LAM 模型来建模互联网人类动作视频当前帧和历史帧之间的隐式变化，然后使 Latent Planner 预测这些变化以进行学习，从而将异构数据源中真实世界的动作知识迁移到模型能力中。 Action Expert（动作专家）：AE 是 MoE 中的另一组专家，通过采用扩散模型，在去噪过程中实现低层级的精细动作序列生成。Action Expert 结构上与Latent Planner 类似，与VLM主干网络共享 Transformer 结构但使用独立 FFN 和投影矩阵
